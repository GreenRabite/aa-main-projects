<h2><a href="#learning-goals" aria-hidden="true" class="anchor" id="user-content-learning-goals"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Learning Goals</h2>
<ul>
<li>Be able to write migrations with indices and constraints
<ul>
<li>Know how to fix the effects of incorrect migrations</li>
</ul>
</li>
<li>Be able to write associations</li>
<li>Know how to seed a project's database</li>
<li>Be able to write custom validations in the model</li>
<li>Be able to solve the N+1 query problem</li>
</ul>
<p>Make sure to refer to the <a href="http://api.rubyonrails.org/classes/ActiveRecord/QueryMethods.html" rel="nofollow">ActiveRecord documentation</a> and this <a href="http://guides.rubyonrails.org/active_record_querying.html" rel="nofollow">ActiveRecord query guide</a> for help with writing your queries.</p>
<p>Generate a new rails project called <code>PollsApp</code>. Refer to yesterday's instructions if you need a reminder. Don't forget to include a PostgreSQL flag:  <code>rails new polls_app --database=postgresql</code>.</p>
<h2><a href="#schema" aria-hidden="true" class="anchor" id="user-content-schema"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Schema</h2>
<p>You need to implement the following schema. Try drawing it out first to
visualize how the models interact with each other. Ask a TA if you have
any questions at this point. Then go ahead and create your migration.
Don't forget to add appropriate indices and constraints!</p>
<ul>
<li><code>User</code>
<ul>
<li>Record a <code>username</code>; make sure it is unique.</li>
</ul>
</li>
<li><code>Poll</code>
<ul>
<li>A <code>Poll</code> belongs to an author (<code>User</code>)</li>
<li>Record a <code>title</code>.</li>
</ul>
</li>
<li><code>Question</code>
<ul>
<li>A <code>Poll</code> has many <code>Question</code>s. Record the <code>text</code>.</li>
</ul>
</li>
<li><code>AnswerChoice</code>
<ul>
<li>A <code>Question</code> has many <code>AnswerChoice</code>s. These are the options that a <code>User</code> can choose from when responding to the question. Record the <code>text</code>.</li>
</ul>
</li>
<li><code>Response</code>
<ul>
<li>A <code>User</code> answers to a <code>Question</code>s by choosing an <code>AnswerChoice</code>.</li>
<li>What pair of foreign keys will you need?</li>
</ul>

<h4><a href="#step-1-responsesibling_responses" aria-hidden="true" class="anchor" id="user-content-step-1-responsesibling_responses"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Step 1: <code>Response#sibling_responses</code></h4>
<p>We'll write a method <code>Response#sibling_responses</code>. This should return
all the other Response objects for the same Question. To do this, first
add the following associations:</p>
<h5><a href="#responsequestion" aria-hidden="true" class="anchor" id="user-content-responsequestion"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>Response#question</code></h5>
<p>This is a <code>has_one through:</code> association. <code>has_one through:</code> works
exactly like <code>has_many through:</code> (it has <code>through</code> and <code>source</code>
options). The only difference is it returns a single object (or <code>nil</code>)
instead of an Array (or empty array).</p>
<h5><a href="#questionresponses" aria-hidden="true" class="anchor" id="user-content-questionresponses"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>Question#responses</code></h5>
<p>This is a <code>has_many through:</code> association.
You've got this :-)</p>
<h5><a href="#singling-yourself-out" aria-hidden="true" class="anchor" id="user-content-singling-yourself-out"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Singling Yourself Out</h5>
<p>Having written these associations, we should be able to write
<code>Response#sibling_responses</code> by (1) calling <code>#question</code> and then (2)
calling <code>#responses</code> on the question.</p>
<p>But wait... won't the current response be included in
<code>response.question.responses</code>? I'm not my own sibling, and neither
should a response be its own sibling.</p>
<p>The answer is yes <em>and</em> no. Since <code>#responses</code> is issuing a query, if
the original response has not been saved to the DB, <code>#sibling_responses</code>
will not contain it. However, once it's persisted to the DB, the query
<em>will</em> return it. <strong>Test this out</strong> by building a new (unsaved) record
and calling <code>sibling_responses</code> and then saving it and calling
<code>sibling_responses</code> again.</p>
<p>This be not good. How can we fix this?</p>
<p>Remember how associations are actually lazy-loading chain-able query
objects? Let's chain a <code>where</code> clause on here to filter out responses
with the same id as <code>self.id</code>.</p>
<p><strong>NB:</strong> Don't forget that you need to use <code>where.not</code> here
because of <a href="/appacademy/curriculum/blob/master/sql/readings/sql-ternary-logic.md">SQL ternary logic</a>.</p>
<h4><a href="#step-2-not_duplicate_response" aria-hidden="true" class="anchor" id="user-content-step-2-not_duplicate_response"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Step 2: <code>not_duplicate_response</code></h4>
<p>Next use <code>Response#sibling_responses</code> to write <code>Response#respondent_already_answered?</code>.
This is a predicate method that checks to see if any sibling
<a href="http://apidock.com/rails/ActiveRecord/FinderMethods/exists%3F" rel="nofollow"><code>exists?</code></a> with the same <code>respondent_id</code>.</p>
<p>Now you can (finally) implement your validation. Use
<code>#respondent_already_answered?</code> to write a <a href="/appacademy/curriculum/blob/master/sql/readings/custom-validations.md">custom
validation</a> method that will ensure that the
respondent has not already answered it.</p>
<h3><a href="#author-cant-respond-to-own-poll" aria-hidden="true" class="anchor" id="user-content-author-cant-respond-to-own-poll"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Author Can't Respond To Own Poll</h3>
<p>Enforce that the creator of the poll must not answer their own
questions: don't let the creator rig the results!</p>
<p>The simplest way is to use associations to traverse from a <code>Response</code>
object back to the <code>AnswerChoice</code>, to the <code>Question</code>, and finally the
<code>Poll</code>. You can then verify whether the poll's author is the same as
the <code>respondent_id</code>. This may involve multiple queries, but we will
later improve this.  <strong>Don't</strong> spend too much time trying to refactor the
method right now.  Many students have gone down this deep, dark rabbit hole
and you'll have time to explore it later.</p>
<p><strong>NB</strong> If you are having trouble with this portion, read the following. Don't
read it until you've attempted it yourself!!</p>
<p>If you are creating a <code>through</code> association that uses another <code>through</code>
association, there is a <a href="https://github.com/rails/rails/issues/20827">Rails bug</a> that makes this return <code>nil</code> if
the record is unsaved. Even Rails has its own list of bugs! To fix this,
explicitly use <code>response.question.poll</code>. You might ask, why has Rails not fixed
this yet? It is extremely rare to make such tenuous associations in real app
development, so it hasn't been a priority. You'll come to see that there are
bugs and issues in all code, even in your operating system and the
implementations of Ruby! There is a limited amount of developers and time.</p>

<p>Improve <code>Response#does_not_respond_to_own_poll</code> to do one query by

<p><code>Response#sibling_responses</code> makes two queries: first it runs a query
to return the <code>Question</code> for the <code>Response</code>, then it performs a second
